# Home Day Swipe TODO

## Goal

Users can horizontally swipe the Home foredrop timeline to move to the previous or next day. The entire Home day surface updates together: date chrome, task timeline, calendar events, habit rows, mixed agenda rows, empty states, and Back to Today visibility.

## Grounding

- PRD: Home is Tasker's single-glance day command center combining tasks, fixed commitments, routines, busy blocks, open gaps, and guidance.
- Calendar docs: Home and timeline should consume the same resolved calendar state; calendar data is read-only and must refresh when the selected day changes.
- Habit docs: Home habit rows are first-class day projections, and visual/computed habit state must not diverge.
- Current UI: `HomeForedropView` hosts the timeline surface and already calls `viewModel.selectDate(_:)` from the week strip.
- Current state: `HomeViewModel.selectedDate`, `HomeChromeSnapshot.selectedDate`, `HomeCalendarSnapshot.selectedDate`, and `HomeTimelineSnapshot.selectedDate` already exist.
- Current top chrome: `HomeChromeSnapshot+Presentation` already shows Back to Today for non-today custom dates.
- Current performance path: `HomeRenderTransaction` commits chrome/tasks/habits/calendar/overlay slices separately through store objects.

## Risks

- `refreshDueTodayAgenda` currently uses `self.selectedDate` inside async work; rapid swipes can apply the wrong habit projection unless the target day is captured.
- `GetHomeFilteredTasksUseCase` already caches by encoded `HomeListScope`, but adjacent-day prefetch should not bloat the unbounded result cache.
- Calendar snapshot currently exposes `eventsTodayCount`; selected-day UI should use selected-day counts where applicable.
- A full-surface `DragGesture` can fight vertical `ScrollView` gestures unless axis-gated carefully.
- The current data stack in this repo is Core Data repositories, not SwiftData. SwiftData constraints still apply if/when this feature is ported, but this implementation should follow the existing Core Data read-model contracts.

## Acceptance Criteria

- Swiping left on the Home foredrop changes to the next day; swiping right changes to the previous day.
- Date chrome, timeline tasks, calendar events, habits, mixed agenda rows, and empty states all reflect the selected day.
- Back to Today appears after any non-today swipe and returns the surface to today.
- Existing week-strip date selection and date picker behavior still work.
- Vertical timeline scrolling and the foredrop reveal handle still feel natural.
- Rapid repeated swipes do not show stale habit/calendar/task data from an intermediate day.
- Reduce Motion users do not receive a large horizontal slide.
